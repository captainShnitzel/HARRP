# Security and safety reporting

HARRP combines networked embedded systems with high-torque electromechanical hardware.

Do not publish:

- active network credentials
- private IP, personal or participant data
- laboratory access details
- unpublished research datasets
- safety bypasses presented without clear warnings

Before testing firmware changes:

1. Isolate or disable actuator power where possible.
2. Secure the arm against uncontrolled movement.
3. Confirm emergency-stop access.
4. Validate communication timeout and motor-enable state.
5. Test one subsystem at a time.
